Центральный репозиторий содержет две главные ветки существующие всё время:
  master 
  develop
Спомогательные ветки
  Ветки функциональности (Feature branches)
  Ветки релизов          (Release branches)
  Ветки исправлений      (Hotfix branches)

feature branch  or tipic branch

  $ git checkout -b myfeature develop  
   
  создание ветки функциональности
  ветки функциональности обычно существуют в репозиториях разрабочиков, но не в главном репозитории
  при начале работы над новой функциональностью делается ответление от ветки разработки (develop)
  Правила слияния:

  $ git checkout develop            переходим на ветку develop для внедрение нового функционала
  $ git merge --no-ff myfeature     
    флаг --no-ff                    создает новый коммит при слиянии, и группирует все внесённые изменения
                                    что позволяет не тирять информациию
  $ git branch -d myfeature         удаление локальной ветки с именем myfeature
  $ git branch --delete myfeature   синоним команды выще. Удалить ветку на которой находишся нельзя
  $ git push origin develop
