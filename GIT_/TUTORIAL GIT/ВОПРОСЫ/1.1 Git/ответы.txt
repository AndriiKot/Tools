1. GIT - это система контроля версий.
   Хранилеше или база данных истории разработки проэкта которая содержит все версии его файлов и каталогов 
   как текущих так уже старых не актуальных.
2. Git 
       1. можно разместить свои начальные файлы
       2. можно редоктировать файлы и добавить эти изменения в git
       3. можно востоновить преведущую  версию проекта

   Git позволяет обмениватся историей разработки

3. Для работы с git не требуется центральный сервер
   У каждого разрабочика стоит локальнная копия git которая хранит историю разработки проэкта

4. Git Server это центральный пункт обменна данными.
   Но все основные опереции выполняются локально.
   Вкачестве git сервера используют в основном github

5. В реальных проэктах как правило делают центарльный серавер
   Это сделанно для удобства 
   На него разрабочики отправляют все изменения, а также скачать изменения которые сделали
   колеги.

6. Git проверяет все данные проэкта (вычесляет контрольную сумму проэкта)
   так как git распределённая система то при повреждении или утери данных у одного разрабочика
   можно их востоновить из истории колеги.
