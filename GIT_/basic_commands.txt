CMD:

cmd             (вход в командную строку)
exit		(выход из командной строки)
cls             (очистить экран)

                но удобней заходить в командную строку через программу Far 
Ctrl + O        вход и выход в командную строку(как же доступны ком. cls exit)

dir             (показывает список текущих папок в данной директории)
cd ..           (поднятия на уровень выше)
cd              (переходит в указанную директорию)
md              (создание папки)
rd (rmdir) /s   (полное удаление каталога со всем его содержимым с подтверждением)
rd /s/q         (полное удаление каталога со всем его содержимым без подтверждением)
                Пример: rd /s/q c:\projext_1


git --version 
git -v          (просмотр текущее git версии)

git --help  
git -h          (выводить список команд которые доступны в git, а также их описание)

git  init       (создаётся файл git)

git status      (выводит информацию о текущем состоянии git)

git add         (добавляет данную папку или в файл  в индекс  git)
git add .       (добавляет в индекс все изменения за которыми следит git )

git rm --cached (удаляем из индекса данную папку или файл)

git commit     
git commit --message "" (фиксирует изменения в git с комментарием(c одинарными кавычками не работает))
git commit -m ""        (сокращенная запись предыдущей команды(с одинарными не работает))

.gitignore              (хранилище файлов и папок которые будут игнорироваться git)

git branch              (показывает на какой ветки проекта мы находимся)
git branch test         (создаёт параллельную ветку проекта c именем test)
git branch -D test      (удаляет ветку с именем test)

git checkout test       (переключает на параллельную ветку с именем test)
                        (при сохранении и в следующем комментировании в параллельной
                         ветки, другие ветки не будут изменены)
git checkout -b test    (сокращенная запись.Это команда создаёт новую ветку с именем "test" , a
                         также сразу переходит на неё)

git merge test          (команда для слияния веток 'master' с 'test'.Что-бы ветки слились нужно перейти на
                         главную ветку "master". )

git config --global user.name
git config --global user.email

копируем ссылку git remote
и вставляем данную строку в командную строку, затем нажимаем Enter

git push -u origin master  (связывает наш локальный проект с github)

git clone http...          (переносит проект с сайта GitHub на локальный компьютер)

git push                   (отправляет изменения на GitHub в том случае когда проект уже создан)

git pull                   (забрать изменения с сайта GitHub (не нужно скачивать сайта или клонировать))

git reset --hard @{u}      возвращает github версию 

git diff              показывает в командной строке какие изменения были введены 
git diff ... (cmd> q) выход из редактора git
